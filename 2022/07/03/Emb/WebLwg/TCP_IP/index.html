网络协议基础

TCP/IP协议模型

Transmission Control Protocol and Internet Protocol.互联网传输过程中用到的一组协议族和互联网模型，协议是促使事物正常运作的一套规则；

数据通信标准分为两类：事实的和法定的

事实标准：未经组织团体承认但己在应用中被广使用和接受的就是事实标准（de facto standard)
法定标准：由官方认可的团体制定的标准称为法定标准(de jure standard) 例如 ISO IEEE IEFT

IETF产生两种文件Internet Draft; RFC(Request For Comments);

分成模型作用

网络通信的过程很复杂
为了降低网络设计的复杂性，将协议讲行了

分层设计的意义
简化相关网络操作
解决异种网络互连时所遇到的兼容性它的最大优点是将服务、接口和协最大优点是将服务，接口，协议明确地区分开来
服务：某一层为上一层提供一些什么功能
接口：上一层如何使用下层的服务
协议：如何实现本层的服务

  1. 物理层(硬件间怎么通讯)

主要定义物理设备标准，如网线的接口类型光纤的接口类型、各种传输介质的传输速率等。它的主要作用是传输比特流，也就是我们常说的数模转换与模数转换，这一层的数据叫做比特；

  1. 数据链路层(数据如何保证正确)

通过各种控制协议，将有差错的物理信道变为无差错的、能可靠传输数据帧的数据链路；

  1. 网络层(数据从一个网络传输到另一个网络)

例如局域网之间不同设备之间发送数据的路径选择即路由的功能；

  1. 传输层(保证数据报文的正确传输)

向用户提供可靠的、端到端的差错和流量控制，保证报文的正确传输。提供建立、连接和拆除传输连接的功能。传输层在网络层基础上，提供“面向连接”和“面向无连接”两种服务

  1. 应用层(协调计算机之间的语言，让用户使用)

会话层，建立和管理应用程序之间的通信。
表示层，解决不同系统之间的通信语法问题
应用层，例如写个android程序，web程序去实现需求。

应用层OSI参考模型 各层解释 TCP/IP模型
应用层 为应用程序提供服务 应用层
表示层 数据格式转化、数据加密
会话层 建立、管理和维护会话
传输层 建立、管理和维护端到端的连接 传输层
网络层 IP选址及路由选择 网络层
数据链路层 提供介质访问和链路管理 接口层
物理层 物理层

HTTP HTTPS

超文本传输协议HTTP(Hyper Text Transfer Protocol)

  1. HTTP
    HTTP是一个客户端和服务器端请求和应答的标准(TCP)，应答的服务器上存储着资源，比如HTML文件和图像。应答服务器为源服务器(origin server)， HTTP客户端发起一个请求，建立一个到服务器指定端口（默认是80端口)的TCP连接。HTTP服务器则在那个端口监听客户端发送过来的请求。一旦收到请求，服务器（向客户端）发回一个状态行

  • URL(统一资源定位符)(http://user:pass@www.example.jp:80/dir/index.htm?uid=1#ch1)

    协议方案名 登录信息(认证) 服务器地址 服务器端口号 带层次的文件路径 查询字符串 片段标识符
    http: user:pass www.example.jp 80 /dir/index.htm uid=1 ch1

    无状态：协议对客户端没有状态存储
    无连接：HTTP/1.1之前，每次请求需要通过TCP三次握手四次挥手，和服务器重新建立连接
    基于请求和响应：由客户端发起请求，服务端响应简单快速、灵活
    通信使用明文、无法保护数据的完整性

  1. HTTPS

    基于HTTP协议，通过SSL或TLS提供加密处理数据、验证对方身份以及数据完整件保护内容加密：采用混合加密技术，中间者无法直接查看明文内容验证身份：通过证书认证客户端访问的是自己的服务器保护数据完整性：防止传输的内容被中间人冒充或者算改

DNS

DNS（Domain Name System）“域名系统”，是一种组织成域层次结构的计算机和网络服务命名系统，它用于TCP/IP网络，它所提供的服务是用来将主机名和域名转换为IP地址的工作，即将网址转换为IP地址

  • Windows 本地DNS文件地址-C:\Windows\System32\drivers\etc\hosts

TCP/UDP的区别

TCP数据流协议，UDP数据包协议
UDP为应用程序提供面向无连接的服务，传输数据之前源端和目的端不需要建立连接，不需要维护连接状态，收发状态等，因此服务器可同时向多个客户端传输相同的消息，UDP适用于对传输效率要求高的运用

TCP UDP

传输可靠性 可靠 不可靠
应用场合 传输大量的数据 传输效率高
速度 慢 快

网络层

点到点的通讯

IP(IPv4、IPv6）相当于OSI参考模型中的网络层
网络层的主要作用是“实现终端节点之间的通信，也叫点对点（end-to-end）通信
IP大致分为三大作用模块，它们是IP寻址、路由（最终节点为止的转发）以及IP分包与组包

IP地址

IP地址唯一标识一台网络设备，由32个二讲制仿组成
IP地址采用点分十讲制格式显示

IP地址-网络地址+主机地址

私有IP地址10.0.0.010.255.255.255
172.16.0.0172.31.255.255
192.168.0.0~192.168.255.255

撞码（masking）区分网络部分和主机部分
掩码使用与IP地址一样的格式
掩码的网络部分和子网部分全都是1，主机部分全都是0

IP路由原理

路由控制（Routing）是指将分组数据发送到最终目标地址的功能。即使网络非常复杂，也可以通过路由控制确定到达目标地址的通路。一旦这个路由控制的运行出现异常，分组数据极有可能“迷失”，无法到达目标地址。因此，一个数据包之所以能够成功地到达最终的目标地址，全靠路由控制

路由器中时刻维持着一张路由表，所有报文的发送和转发都通过查找路由表从相应端口发送，路由表可以是静态配置的，也可以是动态路由协议产生的，路由表包含目的地址-网络掩码-输出接口-下一跳

数据链路层(逐跳传递)

数据链路层的主要功能是完成帧发送和帧接收，包括负责对用户数据进行帧的组装与分解，随时监测物理层的信息监测标志，了解信道的忙闲情况，实现数据链路的收发管理，使用MAC地址进行通讯

CMD查看ipconfig/all-物理地址

共享介质型网络指由多个设备共享一个通信介质的一种网络。最早的以太网和FDDI就是介质共享型网络。在这种方式下，设备之间使用同一个载波信道进行发送和接收。为此，基本上采用半双工通信
共享介质型网络中有两种介质访间控制方式：一种是争用方式，另一种是令牌传递方式

非共享介质网络是指不共享介质，是对介质采取专用的一种传输控制方式，在这种方式下，网络中的每个站直连交换机，由交换机负责转发数据顿。此方式下，发送端与接收端并不共享通信介质，因此很多情况下采用全双工通信方式，如交换式以太网

无线通信通常使用电磁波、红外线、激光等方式进行传播数据。一般在办公室的局城网范围内组成的较高速的连接称为无线局域网

物理层

物理层通过把上层的比特流（0、1的二进制流）转换为电压的高低、灯光的闪灭等物理信号，将数据传输出去。而接收端收到这些物理的信号以后在将这些电压的高低、灯光的闪灭恢复为比特流（0、1的二进制流），即调制与解调的过程，例如Modem。物理层的规范中包括比特流转换规则、缆线结构和质量以及接口形状等

  • 机械特性-网络设备与传输介质之间连接器的外形和尺寸、接口引线数目、引线排列方式、固定和锁定装智
  • 功能特性-规定接口上各种信号线以及各种电平出现的先后顺序，即哪根信号线先动作
  • 电器特性-接口引线上信号电平表示，什么电平代表1，什么电平代表0
  • 规程特性-规定接口某一条引线上出现某一电平表示的含义

IP分片

接收端根据分片报文中的标识符（Identification），标志（Flags），及片偏移（Fragment Offset）字段对分片报文进行重组.
标识符用于识别属于同一个数据包的分片，以区别于同一主机或其他主机发送的其它数据包分片，保证分片被正确的重新组合
标志字段用于判断是否已经收到最后一个分片
片偏移字段表示每个分片在原始报文中的位置。第一个分片的片偏移为0，第二个分片的片偏移表示紧跟第一个分片后的第一个比特的位置

TTL值(Time To Live)

  • 报文在网段间转发时，如果网络设备上的路由规划不合理，就可能会出现环路，导致报文在网络中无眼循环，无法到达目的端。环路发生后，所有发往这个目的地的报文都会被循环转发，随着这种报文逐渐增多，网络将会发生拥塞，没经过一个设备TTL值减一，减为零时丢弃数据包

Version Header Length Total Length Header Checksum
版本号(Ipv4 4) 头部长度(四位最大 0xF) 总长度(最大 65535)，超过MTU(1500)值即分片 头部校验和 IP
  • 服务类型（Type Of Service，TOS）：4位TOS字段分别表示最小延时、最大吞吐量、最高可靠件、最小费用，其中最多有一个能置为1。应用程序根据实际需要来设置TOS值，如ssh和telnet这样的登录程序需要的是最小延时的服务，文件传输ftp需要的是最大吞吐量的服务

ICMP ARP协议

ICMP

  • 用来传递差错，控制，查询等信息
  • Tracert

ARP

  • 当网络设备有数据要发送给另一台网络设备时，必须要知道对方的网络层地址（即IP地址）IP地址由网络层来提供，但是仅有IP地址是不够的，IP数据报文必须封装成帧才能通过数据链路进行发送，数据帧必须包含的MAC地址，因此发送端还必须获取到目的MAC地址，通过目的IP地址来获取目的MAC地址的过程是由端ARP（Address Resolution Protocol）协议来实现的
  • 是解决目的IP与目的MAC关系的
  • 代理ARP
    同一网段，目标IP的MAC
    跨网段且有网关时，网关MAC
    跨网段无网关时，代理ARP
